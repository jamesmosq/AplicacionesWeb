Formatos de imágenes para web

Elegir el formato de imagen correcto es esencial para optimizar el rendimiento y la calidad visual de tu sitio web.

Formato Extensión Compresión Transparencia Animación Mejor para
JPEG .jpg, .jpeg Con pérdida No No Fotografías, imágenes con muchos colores
PNG .png Sin pérdida Sí No Gráficos, logos, imágenes con transparencia
GIF .gif Sin pérdida Sí (binaria) Sí Animaciones simples, imágenes con pocos colores
SVG .svg Sin pérdida Sí Sí (con CSS/JS) Iconos, logos, ilustraciones, gráficos simples
WebP .webp Con/sin pérdida Sí Sí Alternativa moderna para JPEG y PNG
AVIF .avif Con/sin pérdida Sí Sí Formato de próxima generación, mejor que WebP

JPEG (Joint Photographic Experts Group)

JPEG es el formato más común para fotografías en la web debido a su buena compresión y compatibilidad universal.

Características principales:

  • Compresión con pérdida: Reduce el tamaño del archivo eliminando datos de imagen, lo que puede afectar la calidad.
  • Millones de colores: Ideal para fotografías y gradientes.
  • Sin transparencia: No admite áreas transparentes.
  • Sin animación: No permite crear GIFs animados.

Mejor para:

  • Fotografías
  • Imágenes con muchos colores y detalles
  • Banners e imágenes de fondo

Desventajas:

  • La compresión puede causar pérdida de calidad visible (artifacts)
  • No es adecuado para gráficos con texto
  • No soporta transparencia

PNG (Portable Network Graphics)

PNG fue creado como una alternativa libre de patentes a GIF, ofreciendo mejor compresión y más características.

Características principales:

  • Compresión sin pérdida: Mantiene toda la calidad de la imagen original.
  • Transparencia alfa: Soporta transparencia parcial (no solo transparente/opaco).
  • Millones de colores: No tiene limitaciones de color como GIF.
  • Sin animación: No permite crear animaciones.

Mejor para:

  • Logos, íconos y gráficos
  • Imágenes que requieren transparencia
  • Screenshots y gráficos con texto

Desventajas:

  • Archivos más grandes que JPEG para fotografías
  • No soporta animación

GIF (Graphics Interchange Format)

GIF es un formato antiguo pero ampliamente utilizado, especialmente para animaciones simples.

Características principales:

  • Animación: Puede contener múltiples cuadros para crear animaciones.
  • Transparencia binaria: Solo transparente u opaco, sin valores intermedios.
  • Limitado a 256 colores: No es adecuado para fotografías realistas.
  • Compresión sin pérdida: No degrada la calidad de la imagen.

Mejor para:

  • Animaciones simples
  • Ilustraciones con pocos colores
  • Imágenes con áreas de color sólido

Desventajas:

  • Limitado a 256 colores por cuadro
  • Las animaciones pueden tener archivos grandes
  • Transparencia limitada (solo binaria)

SVG (Scalable Vector Graphics)

SVG es un formato basado en XML para gráficos vectoriales, que se mantienen nítidos a cualquier tamaño.

Características principales:

  • Gráficos vectoriales: Basado en formas matemáticas, no en píxeles.
  • Escalable infinitamente: Se ve nítido a cualquier tamaño.
  • Editable con código: Se puede modificar con CSS y JavaScript.
  • Animable: Se puede animar con CSS o JavaScript.
  • Accesible: El texto dentro de SVG es seleccionable y accesible.

Mejor para:

  • Logos e iconos
  • Ilustraciones y gráficos simples
  • Gráficas e infografías
  • Animaciones basadas en vectores

Desventajas:

  • No adecuado para fotografías
  • Puede tener problemas de rendimiento con gráficos muy complejos
  • Compatibilidad limitada con navegadores muy antiguos

WebP

WebP es un formato moderno desarrollado por Google que ofrece mejor compresión que JPEG y PNG.

Características principales:

  • Compresión superior: 25-35% más pequeño que JPEG con la misma calidad visual.
  • Soporte para transparencia: Similar a PNG pero con archivos más pequeños.
  • Soporte para animación: Alternativa más eficiente a GIF.
  • Compresión con y sin pérdida: Versatilidad para diferentes necesidades.

Mejor para:

  • Reemplazo moderno para JPEG, PNG y GIF
  • Sitios web que priorizan la velocidad de carga
  • Fotografías, gráficos y animaciones

Desventajas:

  • No compatible con navegadores antiguos (aunque se puede usar con fallbacks)
  • No tan ampliamente aceptado como JPEG o PNG en software de edición
