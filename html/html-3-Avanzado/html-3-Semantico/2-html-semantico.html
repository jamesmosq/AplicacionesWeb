Introducción al HTML Semántico

El HTML semántico es la práctica de utilizar etiquetas HTML que transmiten significado sobre el contenido que contienen, en lugar de simplemente definir su apariencia.

Esto facilita la comprensión del código tanto para desarrolladores como para tecnologías asistivas y motores de búsqueda.

Etiquetas Semánticas Principales

<header>

Define la cabecera de una página o sección. Normalmente contiene el logotipo, título y navegación principal.

<nav>

Sección dedicada a los enlaces de navegación principales del sitio.

<main>

Contiene el contenido principal de la página. Debe ser único en el documento.

<article>

Representa contenido autónomo que podría distribuirse o reutilizarse independientemente, como un post de blog o un artículo de noticias.

<section>

Representa una sección genérica de contenido, normalmente con un encabezado.

<footer>

Define el pie de página de una sección o de toda la página web.

Importancia para la Accesibilidad Web

El HTML semántico mejora significativamente la accesibilidad web por varias razones:

  • Los lectores de pantalla pueden interpretar mejor la estructura de la página.
  • Facilita la navegación mediante teclado al proporcionar puntos de referencia claros.
  • Ayuda a los usuarios con discapacidades cognitivas a entender mejor la organización del contenido.
  • Permite que las tecnologías asistivas identifiquen correctamente cada parte de la página.

Mejoras en SEO mediante HTML Semántico

El HTML semántico también beneficia el posicionamiento en buscadores (SEO):

  • Los motores de búsqueda comprenden mejor la estructura y el propósito del contenido.
  • Mejora la relevancia del contenido para búsquedas específicas.
  • Facilita la indexación correcta de las diferentes secciones de la página.
  • Ayuda a los algoritmos a identificar el contenido principal versus el secundario.

Uso de <figure> y <figcaption>

Las etiquetas <figure> y <figcaption> son especialmente útiles para contenido ilustrativo:

La etiqueta <figure> representa contenido autocontenido, normalmente una imagen, diagrama o fragmento de código, mientras que <figcaption> proporciona una descripción o título para ese contenido.

Estas etiquetas mejoran la accesibilidad al asociar explícitamente una imagen con su descripción, permitiendo a los lectores de pantalla transmitir esta relación a los usuarios.
