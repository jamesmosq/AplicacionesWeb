¿Qué es el HTML Semántico?

El HTML semántico es el uso de etiquetas HTML que refuerzan el significado de la información en las páginas web, en lugar de simplemente definir su apariencia.

HTML No Semántico

<div class="header">
  <h1>Mi Sitio Web</h1>
</div>

<div class="nav">
  <ul>
    <li><a href="#">Inicio</a></li>
  </ul>
</div>

<div class="content">
  <div class="article">
    <h2>Mi Artículo</h2>
    <p>Contenido...</p>
  </div>
</div>

<div class="footer">
  <p>Copyright 2025</p>
</div>

HTML Semántico

<header>
  <h1>Mi Sitio Web</h1>
</header>

<nav>
  <ul>
    <li><a href="#">Inicio</a></li>
  </ul>
</nav>

<main>
  <article>
    <h2>Mi Artículo</h2>
    <p>Contenido...</p>
  </article>
</main>

<footer>
  <p>Copyright 2025</p>
</footer>

Con HTML semántico, cada elemento de la página tiene un significado específico que describe el tipo de contenido que contiene. Esto hace que el código sea más claro y facilita que los navegadores, motores de búsqueda y tecnologías de asistencia interpreten correctamente el contenido.

Etiquetas Semánticas Principales

La etiqueta <header>

Representa el encabezado introductorio de una página o sección. Normalmente contiene:

  • El logotipo del sitio
  • Título principal (h1)
  • Tagline o descripción breve del sitio
  • A veces incluye la navegación principal

Ejemplo práctico:

<header>
    <img src="logo.png" alt="Mi Logo">
    <h1>Nombre del Sitio</h1>
    <p>Descripción breve del sitio</p>
</header>

La etiqueta <nav>

Define una sección que contiene enlaces de navegación principales. Puede haber varios elementos <nav> en una página.

Ejemplo práctico:

<nav>
    <ul>
        <li><a href="index.html">Inicio</a></li>
        <li><a href="servicios.html">Servicios</a></li>
        <li><a href="contacto.html">Contacto</a></li>
    </ul>
</nav>

La etiqueta <main>

Especifica el contenido principal de la página. Solo debe haber un elemento <main> por documento.

Ejemplo práctico:

<main>
    <h2>Título de la página</h2>
    <p>Contenido principal...</p>
    
    <article>
        <!-- Artículo 1 -->
    </article>
    
    <article>
        <!-- Artículo 2 -->
    </article>
</main>

La etiqueta <article>

Representa contenido independiente y autocontenido. Debe tener sentido por sí solo, como una publicación de blog, un comentario o un widget.

Ejemplo práctico:

<article>
    <header>
        <h2>Título del Artículo</h2>
        <p>Publicado el <time datetime="2025-03-10">10 de marzo de 2025</time> por <a href="#">Autor</a></p>
    </header>
    
    <p>Contenido del artículo...</p>
    
    <footer>
        <p>Categorías: <a href="#">Web</a>, <a href="#">HTML</a></p>
    </footer>
</article>

La etiqueta <section>

Define una sección temática de contenido. Generalmente tiene un encabezado y agrupa contenido relacionado.

Ejemplo práctico:

<section>
    <h2>Nuestros Servicios</h2>
    
    <!-- Primer servicio -->
    <div>
        <h3>Diseño Web</h3>
        <p>Descripción del servicio...</p>
    </div>
    
    <!-- Segundo servicio -->
    <div>
        <h3>Desarrollo</h3>
        <p>Descripción del servicio...</p>
    </div>
</section>

La etiqueta <aside>

Representa contenido tangencialmente relacionado con el contenido principal, como barras laterales, notas al margen, etc.

Ejemplo práctico:

<aside>
    <h3>Artículos Relacionados</h3>
    <ul>
        <li><a href="#">Introducción a CSS</a></li>
        <li><a href="#">Fundamentos de JavaScript</a></li>
    </ul>
</aside>

La etiqueta <footer>

Representa el pie de página de una sección o de la página completa. Típicamente contiene información de copyright, enlaces a políticas, mapa del sitio, etc.

Ejemplo práctico:

<footer>
    <p>© 2025 Mi Sitio Web. Todos los derechos reservados.</p>
    <nav>
        <ul>
            <li><a href="politica-privacidad.html">Política de Privacidad</a></li>
            <li><a href="terminos.html">Términos de Uso</a></li>
        </ul>
    </nav>
</footer>

Importancia para la Accesibilidad Web

El HTML semántico es crucial para la accesibilidad web por varias razones:

Beneficios para la accesibilidad

1. Lectores de pantalla

Los lectores de pantalla utilizan las etiquetas semánticas para navegar y describir el contenido a usuarios con discapacidad visual. Por ejemplo:

  • Anuncian "encabezado" cuando encuentran un <header>
  • Permiten saltar directamente al <main> con atajos de teclado
  • Identifican la navegación principal dentro de <nav>
2. Navegación por teclado

Las etiquetas semánticas mejoran la navegación con teclado, permitiendo a los usuarios saltar entre secciones importantes sin tener que tabular por cada elemento.

3. ARIA (Accessible Rich Internet Applications)

Muchas etiquetas semánticas tienen roles ARIA implícitos, lo que significa que no necesitas agregar atributos ARIA adicionales:

Elemento HTML Rol ARIA implícito
<header> banner (cuando es hijo directo de body)
<nav> navigation
<main> main
<aside> complementary
<footer> contentinfo (cuando es hijo directo de body)
4. Modo de lectura de navegadores

Navegadores como Firefox y Safari tienen un "Modo de lectura" que elimina distracciones y reformatea el contenido para una mejor experiencia de lectura. Este modo se basa en etiquetas semánticas para identificar el contenido principal.

Un ejemplo de estructura accesible:

<body>
    <header>
        <h1>Título del Sitio</h1>
    </header>
    
    <nav>
        <!-- Barra de navegación principal con atributo ARIA adicional -->
        <ul aria-label="Navegación principal">
            <li><a href="#">Inicio</a></li>
            <li><a href="#">Servicios</a></li>
        </ul>
    </nav>
    
    <main id="contenido-principal">
        <!-- Un link de salto para usuarios de teclado -->
        <a href="#contenido-principal" class="skip-link">Saltar al contenido principal</a>
        
        <article>
            <h2>Título del Artículo</h2>
            <p>Contenido...</p>
        </article>
    </main>
    
    <aside>
        <h2>Contenido Relacionado</h2>
        <!-- ... -->
    </aside>
    
    <footer>
        <p>Información de copyright</p>
    </footer>
</body>

Mejoras en SEO mediante HTML Semántico

El HTML semántico juega un papel importante en el SEO (Search Engine Optimization) porque ayuda a los motores de búsqueda a entender mejor el contenido de tu página:

Aspecto SEO Beneficio del HTML Semántico
Comprensión del contenido Los motores de búsqueda pueden identificar claramente qué partes de tu página contienen el contenido principal vs. navegación, encabezados, pies de página, etc.
Snippets enriquecidos El uso de etiquetas semánticas como <article> y elementos como <time> puede ayudar a generar fragmentos enriquecidos en los resultados de búsqueda.
Relevancia de contenido El uso adecuado de encabezados jerárquicos (h1-h6) dentro de las secciones semánticas ayuda a establecer la importancia relativa de diferentes partes del contenido.
Indexación móvil Con la indexación móvil primero (mobile-first indexing), Google prioriza sitios bien estructurados con HTML semántico que ofrezcan buena experiencia en dispositivos móviles.

Ejemplo de estructura optimizada para SEO:

<article>
    <header>
        <h1>Título Optimizado con Palabras Clave</h1>
        <p>
            Publicado por <span itemscope itemtype="http://schema.org/Person">
                <span itemprop="name">Nombre del Autor</span>
            </span>
            el <time datetime="2025-03-10" itemprop="datePublished">10 de marzo de 2025</time>
        </p>
    </header>
    
    <section>
        <h2>Primer Subtítulo Relevante</h2>
        <p>Contenido valioso con palabras clave naturales...</p>
        
        <figure>
            <img src="imagen.jpg" alt="Descripción optimizada con palabras clave" itemprop="image">
            <figcaption>Descripción detallada de la imagen con términos relevantes</figcaption>
        </figure>
    </section>
    
    <section>
        <h2>Segundo Subtítulo Importante</h2>
        <p>Más contenido valioso...</p>
    </section>
    
    <footer>
        <p>Categorías: 
            <a href="#" rel="category">Categoría Principal</a>, 
            <a href="#" rel="category">Categoría Secundaria</a>
        </p>
    </footer>
</article>

Uso de <figure> y <figcaption>

Las etiquetas <figure> y <figcaption> son especialmente útiles para contenido ilustrativo como imágenes, diagramas, fragmentos de código, etc., que requieren una leyenda explicativa.

Ejemplo básico con imagen:

Código del ejemplo anterior:

<figure>
    <img src="imagen.jpg" alt="Breve descripción para accesibilidad">
    <figcaption>Fig 1. Esta es una leyenda que describe la imagen y proporciona contexto adicional.</figcaption>
</figure>

Ejemplo con fragmento de código:

Código del ejemplo anterior:

<figure>
    <pre><code>function saludar(nombre) {
    return `¡Hola, ${nombre}!`;
}

console.log(saludar('Mundo'));</code></pre>
    <figcaption>Fig 2. Función simple en JavaScript que saluda al usuario.</figcaption>
</figure>

Ejemplo con múltiples imágenes en un solo figure:

Código del ejemplo anterior:

<figure>
    <div style="display: flex; justify-content: space-between;">
        <img src="imagen1.jpg" alt="Versión anterior" style="width: 48%">
        <img src="imagen2.jpg" alt="Nueva versión" style="width: 48%">
    </div>
    <figcaption>Fig 3. Comparativa entre dos versiones del diseño web.</figcaption>
</figure>

Beneficios de usar figure y figcaption:

  • Accesibilidad: Los lectores de pantalla identifican correctamente la relación entre el contenido y su descripción.
  • SEO: Proporciona contexto adicional para imágenes y otros contenidos visuales.
  • Estructura: Agrupa lógicamente contenido relacionado.
  • Estilo: Facilita aplicar estilos consistentes a todos los elementos ilustrativos.
